# テストケース雛形（サンプル）

本ドキュメントは、演習で作成する「テストケース」の書き方とフォーマットを示すものです。

## 1. テスト観点の洗い出し（例）
テストケースを作る前に、「何を、どのような視点（観点）で確認するか」を整理します。

| カテゴリ | 観点名 | 確認内容の例 |
|---|---|---|
| **正常系** | 基本機能の動作 | ログインできるか、本が借りられるか |
| **正常系** | データ整合性 | 借りた後、在庫数が正しく1減っているか |
| **異常系** | 入力バリデーション | タイトルを空で登録したときにエラーになるか |
| **異常系** | 権限チェック | 一般ユーザーが書籍削除URLに直接アクセスして弾かれるか |
| **境界値** | 貸出上限 | 5冊借りている状態で、もう1冊を拒否されるか |
| **特殊条件** | 日付算出 | 土日に返却期限が重なった際、月曜に延期されるか |

## 2. テストケース一覧フォーマット
演習では、以下の項目を網羅した一覧表を作成してください。

| ID | テスト対象機能 | 確認内容（期待値） | 手順 | 結果 | 備考 |
|---|---|---|---|---|---|
| TC-01 | ログイン | 正しいID/PWでログインできること | 1. `auth/login`へアクセス<br>2. ID `user01` PW `password` を入力<br>3. ログインボタン押下 | OK | |
| TC-02 | 貸出 | 在庫0の書籍は借りられないこと | 1. 在庫0の「リーダブルコード」を探す<br>2. 「借りる」ボタンの状態を確認する | [判定] | ボタンが無効かエラーか？ |
| TC-03 | 貸出上限 | 同時に6冊借りることはできないこと | 1. すでに5冊借りる<br>2. 6冊目の貸出を試みる | [判定] | 5冊制限の確認 |
| TC-04 | 書籍削除 | 貸出中の書籍を削除しようとすると拒否されること | 1. 誰かが借りている本を特定<br>2. 管理者でログインし削除を試みる | [判定] | 整合性の確認 |

### 記入上の注意
- **期待値:** 「〜できること」「〜と表示されること」のように、合格基準を明確に書く。
- **結果:** テスト実施後に「OK / PASS」または「NG / FAIL」を記入する。
- **NGの場合:** 見つけた不具合は GitHub Issue へ報告し、備考欄に Issue番号（例: #5）を記載する。
